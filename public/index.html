Documentação da API de Movimentos

Bem-vindo à documentação da API de Movimentos. Esta API fornece dados de rankings de usuários baseados em seus recordes para cada movimento disponível.

Rotas Disponíveis

1. Obter o ranking de um movimento específico

URL: /movements/{id}/ranking

Método HTTP: GET

Parâmetro:

  • {id} - O ID do movimento que você deseja consultar.
GET /movements/1/ranking
GET /movements/2/ranking
GET /movements/3/ranking
{
        "movimento":"Deadlift",
        "ranking":[
        {
            "posição":1,
            "usuário:"Jose",
            "recorde pessoal":190,
            "data":"06/01/2021"
        },
        {
            "posição":2,
            "usuário":"Joao",
            "recorde pessoal":180,
            "data":"04/01/2021"
        },
        {
            "posição":3,
            "usuário":"Paulo",
            "recorde pessoal":170,
            "data":"03/01/2021"
        }]
    }
  • movimento: O nome do movimento consultado.
  • ranking: Uma lista ordenada dos usuários e seus recordes para o movimento.
  • posição: A posição do usuário no ranking (usuários com o mesmo recorde ocupam a mesma posição).
  • usuario: O nome do usuário que realizou o movimento.
  • recorde pessoal: O maior valor de recorde do usuário.
  • data: A data em que o recorde foi alcançado.
  • 404 - Movimento não encontrado: Se o ID do movimento fornecido não existir.
  • 500 - Erro no servidor: Qualquer erro interno que ocorra ao processar a requisição.

Exemplo de uso com cURL

Você pode testar a API usando a ferramenta cURL no terminal:

curl http://localhost:8080/movements/1/ranking

Você pode testar a API acessando esse link na url:

http://localhost:8080/movements/1/ranking

Você pode testar a API colocando esse link no Postman ou Insominia:

http://localhost:8080/movements/1/ranking
